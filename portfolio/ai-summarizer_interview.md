# ai-summarizer 면접 예상 질문

## 🔥 트러블슈팅 경험

### Q1. 프로젝트 진행 중에 발생한 에러 중에서 가장 어려웠던 에러는 무엇이었나요? 그 에러를 어떻게 해결했나요?
**A1.** 프로젝트 진행 중에 발생한 에러 중에서 가장 어려웠던 에러는 `FileNotFoundError`와 `ValueError`였습니다. 이 에러를 해결하기 위해 `try-except`문을 사용하여 에러를 처리하였습니다. 예를 들어, 파일을 읽어올 때 `FileNotFoundError`가 발생할 수 있으므로, `try` 블록에서 파일을 읽어오고, `except` 블록에서 에러 메시지를 출력하도록 구현했습니다.

### Q2. 프로젝트에서 `401/KeyError` 수정을 위해 어떤 방법을 사용했나요?
**A2.** 프로젝트에서 `401/KeyError` 수정을 위해 에러가 발생하는 코드 부분을 수정하였습니다. 예를 들어, API 요청 시에 인증 정보가 올바르지 않아 `401` 에러가 발생할 수 있으므로, 인증 정보를 확인하고 수정하였습니다. 또한, `KeyError`가 발생할 수 있는 코드 부분을 수정하였습니다. 예를 들어, 딕셔너리에서 키가 존재하지 않을 때 `KeyError`가 발생할 수 있으므로, 키가 존재하는지 확인하고 처리하였습니다.

## 🛠 기술 선택 / 구현

### Q3. 프로젝트에서 Python을 사용한 이유는 무엇인가요?
**A3.** 프로젝트에서 Python을 사용한 이유는 Python이 자연어 처리와 관련된 라이브러리가 풍부하여 텍스트 요약기 구현에 적합하다고 판단했기 때문입니다. 또한, Python은 개발 속도가 빠르고, 코드가 간결하여 빠르게 프로토타입을 개발할 수 있었습니다.

### Q4. 프로젝트에서 Groq API를 사용한 이유는 무엇인가요?
**A4.** 프로젝트에서 Groq API를 사용한 이유는 Groq API가 텍스트 요약기 구현에 필요한 기능을 제공하고, 사용하기 쉽다고 판단했기 때문입니다. 또한, Groq API는 높은 성능을 제공하여 빠르게 요약을 생성할 수 있었습니다.

## 💡 협업 / 회고

### Q5. 프로젝트를 진행하면서 배운 점은 무엇인가요?
**A5.** 프로젝트를 진행하면서 배운 점은 파일을 읽어와서 요약을 생성하는 방법을 배웠습니다. 또한, 에러 처리 방법을 배웠습니다. 예를 들어, `FileNotFoundError`와 `ValueError`를 처리하는 방법을 배웠습니다. 이러한 경험을 통해 더 나은 개발자가 되기 위해 노력하겠습니다.
