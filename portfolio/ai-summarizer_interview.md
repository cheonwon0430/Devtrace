# ai-summarizer 면접 예상 질문

## 🔥 트러블슈팅 경험

### Q1. 개발 중 발생한 에러를 해결한 경험에 대해 설명해 주세요.
**A1.** 오늘 개발 중 "not found"와 "ImportError" 에러가 발생했습니다. 에러 로그를 분석하여 문제의 원인을 파악한 후, 코드에서 잘못된 경로를 수정하고 필요한 모듈을 올바르게 임포트하여 문제를 해결했습니다. 이러한 과정을 통해 에러 로그를 분석하는 능력을 키울 수 있었습니다.

### Q2. 특정 에러를 해결하기 위해 어떤 접근 방식을 사용했나요?
**A2.** 에러를 해결하기 위해 먼저 에러 로그를 확인하고, 문제가 발생한 코드의 위치를 찾아 수정했습니다. "not found" 에러는 파일 경로가 잘못 지정되어 있었고, "ImportError"는 필요한 모듈이 설치되지 않았기 때문에, 해당 모듈을 설치한 후 코드를 수정하여 문제를 해결했습니다. 이 과정에서 문제를 체계적으로 분석하는 방법을 배웠습니다.

## 🛠 기술 선택 / 구현

### Q3. `summarize_file` 함수를 추가한 이유는 무엇인가요?
**A3.** `summarize_file` 함수를 추가한 이유는 파일에서 직접 텍스트를 읽어 요약을 생성할 수 있는 기능을 제공하기 위해서입니다. 이를 통해 사용자가 파일을 업로드하면 자동으로 요약 결과를 얻을 수 있게 되어, 사용자 편의성을 높이고 효율적인 작업이 가능해졌습니다.

### Q4. `FileNotFoundError`와 `ValueError`를 처리한 방법에 대해 설명해 주세요.
**A4.** `FileNotFoundError`는 파일이 존재하지 않을 때 발생하는 에러로, 이를 처리하기 위해 파일이 존재하는지 확인하는 조건문을 추가했습니다. `ValueError`는 잘못된 값이 입력될 때 발생하는 에러로, 입력값을 검증하는 로직을 추가하여 유효하지 않은 값이 들어오지 않도록 했습니다. 이러한 예외 처리를 통해 프로그램의 안정성을 높일 수 있었습니다.

## 💡 협업 / 회고

### Q5. 이번 프로젝트에서 배운 점은 무엇인가요?
**A5.** 이번 프로젝트를 통해 파일을 읽고 요약을 생성하는 방법을 배웠습니다. 또한, 에러를 처리하는 과정에서 코드의 안정성을 높이는 것이 얼마나 중요한지를 깨달았습니다. 협업을 통해 다른 개발자들과의 소통과 피드백이 프로젝트의 품질을 높이는 데 큰 도움이 된다는 것도 느꼈습니다.
