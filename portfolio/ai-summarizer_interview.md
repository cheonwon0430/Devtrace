# ai-summarizer 면접 예상 질문

## 🔥 트러블슈팅 경험

### Q1. 프로젝트에서 발생한 `FileNotFoundError`와 `ValueError`를 어떻게 처리했나요?
**A1.** 프로젝트에서 발생한 `FileNotFoundError`와 `ValueError`를 처리하기 위해, `try-except`문을 사용하여 예외를 캐ッチ하고 적절한 에러 메시지를 출력하도록 구현했습니다. 또한, 파일 경로를 확인하고, 파일이 존재하는지 확인하는 코드를 추가하여 에러를 방지했습니다.

### Q2. 프로젝트에서 발생한 `401/KeyError`를 어떻게 수정했나요?
**A2.** 프로젝트에서 발생한 `401/KeyError`를 수정하기 위해, API 키를 확인하고, 올바른 키를 사용하도록 수정했습니다. 또한, 에러가 발생할 경우에 대한 안전처리를 추가하여, 프로그램이 중단되지 않도록 구현했습니다.

## 🛠 기술 선택 / 구현

### Q3. 프로젝트에서 Python을 사용한 이유는 무엇인가요?
**A3.** 프로젝트에서 Python을 사용한 이유는, Python이 자연어 처리와 머신러닝에 강한 언어이며, 쉽게 구현할 수 있고, 많은 라이브러리가 존재하기 때문입니다. 또한, Groq API를 사용하여 텍스트 요약기를 개발해야 했기 때문에, Python의 간결성과 유연성이 프로젝트에 적합했습니다.

### Q4. 프로젝트에서 `summarize_file` 함수를 추가한 이유는 무엇인가요?
**A4.** 프로젝트에서 `summarize_file` 함수를 추가한 이유는, 파일을 읽어서 요약 생성을 처리해야 했기 때문입니다. 이 함수를 추가하여, 파일을 읽고 요약을 생성하는 기능을 구현할 수 있었으며, 프로젝트의 기능을 강화할 수 있었습니다.

## 💡 협업 / 회고

### Q5. 프로젝트를 진행하면서 배운 점은 무엇인가요?
**A5.** 프로젝트를 진행하면서, 파일을 읽어서 요약 생성을 처리하는 방법을 배웠습니다. 또한, `FileNotFoundError`와 `ValueError`를 처리하는 방법을 배웠으며, 프로젝트를 진행하면서 발생하는 에러를 어떻게 처리해야 하는지 배웠습니다. 이러한 경험을 통해, 더 나은 개발자로 성장할 수 있었습니다.
