# ai-summarizer 면접 예상 질문

## 🔥 트러블슈팅 경험

### Q1. 프로젝트를 진행하면서 어떤 오류를 만났으며, 어떻게 해결하였나요?
**A1.** 프로젝트를 진행하면서 FileNotFoundError와 ValueError를 만났습니다. 이러한 오류를 해결하기 위해, 파일을 읽어오는 부분에서 예외 처리를 추가하여 오류를 해결하였습니다. 또한, 청킹 처리를 통해 KeyError를 안전하게 처리할 수 있도록 하였습니다.

### Q2. 401/KeyError 오류를 어떻게 해결하였나요?
**A2.** 401/KeyError 오류를 해결하기 위해, API 요청 부분에서 인증 정보를 확인하고, 키를 제대로 전달하는지 확인하였습니다. 또한, 오류 로그를 분석하여 에러 패턴을 확인하고, 반복 에러를 감지하여 해결하였습니다.

## 🛠 기술 선택 / 구현

### Q3. 프로젝트에서 Python을 사용한 이유는 무엇인가요?
**A3.** 프로젝트에서 Python을 사용한 이유는, Python이 자연어 처리와 머신러닝에 강점을 가지고 있으며, Groq API를 쉽게 연동할 수 있기 때문입니다. 또한, Python의 간결한 문법과 풍부한 라이브러리가 프로젝트 개발을 빠르게 진행할 수 있도록 도와주었습니다.

### Q4. summarize_file 함수를 추가한 이유는 무엇인가요?
**A4.** summarize_file 함수를 추가한 이유는, 파일을 읽어서 요약 생성을 처리하기 위해서입니다. 이 함수를 통해 파일을 읽어오는 부분과 요약 생성 부분을 분리하여, 코드를 더 모듈화하고 재사용성을 높일 수 있었습니다.

## 💡 협업 / 회고

### Q5. 프로젝트를 진행하면서 배운 점은 무엇인가요?
**A5.** 프로젝트를 진행하면서, 파일을 읽어서 요약 생성을 처리하는 방법과 에러 로그를 분석하여 개발 에러를 확인하는 방법을 배웠습니다. 또한, DevTrace v2.0을 사용하여 개발 일지를 자동으로 생성하는 방법을 배웠습니다. 이러한 경험을 통해, 더 효율적인 개발 방법과 협업 방법을 배울 수 있었습니다.
