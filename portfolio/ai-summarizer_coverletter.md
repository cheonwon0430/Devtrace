### 3. [Tech] 공고 내 '세부 업무'를 참고하여 Platform/Client/Server/Infra 를 관심 순으로 나열하고 (ex. Platform > Client = Server > Infra, 관심도가 유사하다면 모두 '='으로 작성), 각 분야에 관심을 가지게 된 계기 혹은 경험(ex. 스터디, 교육 등)이 있다면 서술해 주세요. (1000자 이내)

Platform > Server > Client > Infra

ai-summarizer 프로젝트를 통해 Platform 분야에 대한 깊은 관심을 가지게 되었습니다. 프로젝트 초기 단계에서 Groq API를 활용하여 텍스트 요약 기능을 구현하는 과정에서, API와의 상호작용이 얼마나 중요한지를 깨달았습니다. 특히, .env 파일을 통해 API 키를 안전하게 관리하는 방법을 배우면서 보안의 중요성을 인식하게 되었습니다. 이는 단순한 기능 구현을 넘어, 시스템의 구조적 안전성을 확보하는 데 필수적인 요소임을 알게 해주었습니다.

Server 분야에서도 많은 경험을 쌓았습니다. 파일 요약 기능을 추가하기 위해 `summarize_file` 함수를 구현하면서, 파일을 읽고 처리하는 과정에서 발생할 수 있는 다양한 에러를 다루는 방법을 배웠습니다. 이 과정에서 `FileNotFoundError`와 `ValueError`를 처리하는 기술을 익혔고, 이는 서버에서의 데이터 처리 안정성을 높이는 데 기여했습니다. 에러를 해결하는 과정에서 어떻게 하면 더 나은 코드 구조를 만들 수 있을지를 고민하게 되었고, 이러한 경험이 Server 분야에 대한 관심을 더욱 증대시켰습니다.

Client 분야는 상대적으로 적은 경험을 가지고 있지만, CSS와 HTML 파일을 수정하면서 사용자 인터페이스의 중요성을 느꼈습니다. 사용자 경험을 개선하기 위해 디자인 요소를 고려하는 것이 얼마나 중요한지를 깨달았습니다. 마지막으로 Infra 분야는 현재로서는 상대적으로 적은 경험을 가지고 있지만, 시스템의 전반적인 안정성을 확보하는 데 필수적이라는 점은 인식하고 있습니다. 앞으로의 경험을 통해 Infra 분야에 대한 이해를 더 깊이 있게 확장해 나가고 싶습니다.
