# 1. 어떤 문제를 해결할 때, 'Why?'를 놓치지 않고 문제의 본질을 찾는 데 집중했던 경험이 있다면 적어주세요. (1000자 이내)

ai-summarizer 프로젝트에서 파일 요약 기능을 개발하면서 'Why?'에 집중한 경험이 있습니다. 처음에 파일을 읽어 요약을 생성하는 과정에서 `FileNotFoundError`와 `ValueError`가 발생했습니다. 단순히 에러를 수정하는 것이 아니라, 왜 이러한 에러가 발생하는지를 분석했습니다. 파일 경로가 잘못되었거나, 파일 형식이 예상과 다를 경우 발생하는 문제라는 것을 깨달았습니다. 이를 통해 단순한 에러 수정이 아닌, 파일을 안전하게 처리하기 위한 구조적 접근이 필요하다는 것을 인식했습니다. 결과적으로 `summarize_file` 함수를 추가하고, 파일을 읽는 과정에서 발생할 수 있는 다양한 예외를 처리하도록 개선했습니다. 이러한 과정에서 단순히 문제를 해결하는 것이 아니라, 문제의 본질을 이해하고 그에 맞는 해결책을 마련하는 것이 얼마나 중요한지를 깨달았습니다. 이 경험은 앞으로 개발자로서 문제를 해결할 때 항상 근본 원인을 찾는 태도를 가지게 해주었습니다.

# 2. 어떤 일을 하면서 '이 정도면 충분하다'는 생각 대신, 더 완성도를 높이고 싶어 끝까지 몰입했던 경험이 있나요? 그때 어떤 점이 가장 도전적이었고, 어떻게 극복하셨는지 적어주세요. (1000자 이내)

ai-summarizer 프로젝트에서 요약 생성 기능을 구현할 때, '이 정도면 충분하다'는 생각을 버리고 완성도를 높이기 위해 몰입했던 경험이 있습니다. 처음에 요약 기능을 구현했을 때, 기본적인 기능은 작동했지만, 긴 텍스트를 처리할 때는 성능이 떨어졌습니다. 그 시점에서 '충분하다'고 느낄 수 있었지만, 저는 더 나은 결과를 원했습니다. 그래서 `chunk_text` 함수를 추가하여 긴 텍스트를 청킹하여 처리하는 방식을 도입했습니다. 이 과정에서 가장 도전적이었던 점은 기존 코드와의 호환성을 유지하면서 새로운 기능을 추가하는 것이었습니다. 이를 위해 기존 코드를 철저히 분석하고, 함수 간의 관계를 명확히 하여 리팩토링을 진행했습니다. 결국, 새로운 기능을 성공적으로 추가하면서도 기존의 안정성을 유지할 수 있었습니다. 이러한 경험은 저에게 완성도에 대한 집착과 끝까지 몰입하는 것이 얼마나 중요한지를 깨닫게 해주었습니다.

# 3. [Tech] 공고 내 '세부 업무'를 참고하여 Platform/Client/Server/Infra 를 관심 순으로 나열하고 각 분야에 관심을 가지게 된 계기 혹은 경험(ex. 스터디, 교육 등)이 있다면 서술해 주세요. (1000자 이내)

Platform > Server > Client > Infra

ai-summarizer 프로젝트를 진행하면서 기술 분야에 대한 관심이 더욱 깊어졌습니다. 특히 Platform 분야에 대한 관심은 Groq API를 활용한 텍스트 요약기 개발 경험에서 비롯되었습니다. API를 통해 데이터를 처리하고, 이를 기반으로 요약 기능을 구현하면서 플랫폼의 중요성을 체감했습니다. Server 분야는 `summarizer.py` 파일을 수정하며 서버와의 데이터 통신 및 에러 처리에 대한 이해가 깊어졌습니다. 특히 `FileNotFoundError`와 `ValueError`를 처리하는 과정에서 서버와 클라이언트 간의 상호작용을 이해하게 되었습니다. Client 분야는 UI 요소를 다루는 `viewer.css`와 HTML 파일 수정 작업을 통해 사용자 경험을 개선하는 데 기여하면서 관심이 생겼습니다. 마지막으로 Infra 분야는 프로젝트 초기 설정과 관련된 작업을 통해 기본적인 인프라의 중요성을 인식하게 되었습니다. 이러한 경험들은 각 분야에 대한 깊은 관심을 불러일으켰고, 앞으로도 계속해서 발전시키고 싶습니다.
